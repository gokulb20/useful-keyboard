## Key Decisions
- **Decision**: Use `AVAudioRecorder.averagePower` instead of `AVAudioEngine` tap or Core Audio
  - **Context**: Need mic audio levels to drive waveform animation in floating indicator
  - **Rationale**: Already recording with `AVAudioRecorder` in `MicrophoneRecorder.swift`. Metering is a single flag flip (`isMeteringEnabled = true`). Apple computes RMS+dB internally — same math as WisprFlow's `He()` but free.
  - **Alternatives rejected**: `AVAudioEngine` (separate audio pipeline, buffer management overhead), Core Audio C API (massive complexity), Web Audio API (requires Electron)

- **Decision**: CALayer sublayers for bar rendering, not SwiftUI
  - **Context**: `FloatingIndicatorController` is entirely CALayer-based (`NSPanel` with `CALayer` styling)
  - **Rationale**: Adding `CAShapeLayer` bars as sublayers keeps same paradigm, GPU-composited, zero SwiftUI hosting overhead for a 7-bar widget
  - **Alternatives rejected**: SwiftUI `NSHostingView` overlay (unnecessary framework crossing for this small component), `NSView.draw()` (CPU-drawn, redraws every frame)

- **Decision**: Two-layer animation approach (WisprFlow pattern)
  - **Context**: User confirmed WisprFlow has gentle wave even in silence, with amplitude scaling on top
  - **Rationale**: `CABasicAnimation(keyPath: "transform.scale.y")` with `autoreverses`/`repeatCount = .infinity` for base oscillation runs on GPU render server (zero main thread cost). `CADisplayLink` polls `averagePower` and adjusts bar `bounds.size.height` for amplitude envelope.

## Lessons Learned
- WisprFlow's waveform is NOT purely CSS-looped — it has real audio reactivity via `AudioWorklet → RMS → IPC → exponential smoothing → threshold bar activation`. The CSS loop is just the idle baseline.
- Handy (Tauri/Rust) takes a cleaner approach: Rust `visualizer.rs` calculates amplitude, sends single float over IPC. No idle animation — dots are purely amplitude-driven.
- Exponential smoothing constant 0.7/0.3 (proven by WisprFlow) prevents jitter while maintaining responsiveness. This is the sweet spot.
- `AVAudioRecorder.averagePower(forChannel:)` returns dB in range roughly -160 to 0. Practical voice range is about -50 to 0 dB. Normalize: `clamp(dB, -50, 0)` → map to `0.0...1.0`.

## Nuances & Edge Cases
- `recorder.updateMeters()` MUST be called before each `averagePower` read — it doesn't auto-update
- `averagePower` returns `-160` (effectively silence) when not recording — safe default
- The `CADisplayLink` callback must be on main thread since `FloatingIndicatorController` is `@MainActor`
- Staggered `beginTime` for base oscillation: use `CACurrentMediaTime() + offset` per bar, not absolute times
- The floating indicator is an `NSPanel` with `level: .floating` — `CABasicAnimation` works fine on panel layers but test with `collectionBehavior: [.canJoinAllSpaces]` to ensure animations don't pause on space switch
